# QA 역할 규칙

## 역할
- 담당: 기능 검증, 오류 판단, 테스트 시나리오 작성 및 검수
- 참고: `.claude/rules/global.md` 를 반드시 함께 적용한다

---

## 기본 원칙
- 오류인지 아닌지 애매할 경우 **반드시 사용자에게 먼저 보고**하고 판단을 구한다.
- 오류가 아니라고 판단한 근거를 명확히 설명한다.
- 재현 가능한 오류만 오류로 판정한다. 한 번만 발생한 경우 재현 시도 후 판정한다.

---

## 오류 판정 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| 🔴 Critical | 핵심 기능 동작 불가 | GPX 업로드 실패, 지도 미표출 |
| 🟠 Major | 기능은 동작하나 결과가 틀림 | 거리 계산 오류, 달성률 오표시 |
| 🟡 Minor | 기능은 정상이나 UX 문제 | 로딩 표시 누락, 에러 메시지 미표시 |
| 🟢 Normal | 오류 아님 | 의도된 동작, 디자인 차이 |

---

## 테스트 시나리오

### 🗺️ 지도 화면
| 항목 | 기대 결과 |
|------|----------|
| GPX 업로드 후 지도 진입 | 러닝 경로 폴리라인 표출 |
| 오늘 토글 선택 | 오늘 날짜 경로만 표출 |
| 일주일 토글 선택 | 최근 7일 경로 표출 |
| 기록 없을 때 | 빈 화면 대신 안내 메시지 표출 |

### 📁 기록 업로드 화면
| 항목 | 기대 결과 |
|------|----------|
| GPX 파일 업로드 | 업로드 성공 + 기록 요약 표출 |
| 잘못된 확장자 업로드 | 에러 메시지 표출 |
| 10MB 초과 파일 업로드 | 용량 초과 에러 표출 |
| 오늘 업로드 완료 시 | 캘린더에 완료 표시 |
| 업로드 중 | 로딩 표시 |

### 🎯 목표치 화면
| 항목 | 기대 결과 |
|------|----------|
| 목표 거리 설정 | 프로그레스 바 업데이트 |
| 목표 달성 시 | 100% 표시 |
| 목표 미설정 시 | 안내 메시지 표출 |

### 🔌 API
| 항목 | 기대 결과 |
|------|----------|
| 전체 기록 조회 | 200 OK + 기록 배열 반환 |
| 존재하지 않는 리소스 조회 | 404 반환 |
| 잘못된 파일 업로드 | 400 반환 |
| 서버 재시작 후 | data.sql 데이터 정상 로드 |

## 보고 형식

오류 발견 시 아래 형식으로 보고한다:

```

- 발생 위치: 
- 재현 방법: 
- 기대 결과: 
- 실제 결과: 
- 판단 근거: 
```
